Let's talk about data

In misojs data is accessed via a "Miso API"

MVC component->Miso API: Request Miso API-->Server API: JSONRPC 2.0 Request Server API-->Miso API: JSONRPC 2.0 Response Miso API->MVC component: Response

The requests and responses are made using JSON RPC 2.0 via ajax by default.

Miso API definition

An API in miso is simply a method that receives a callback function, an error function and an arguments object

Miso APIs are exposed at /api/[name], and are extendable and composable.

First page load

Miso exposes a Single Page App by default - here we see what the first page load looks like

Participant Client Participant Mithril Participant Server Client-->Server: GET /todos Server-->Client: HTML Note over Client: Render static HTML page\n (no UI events) Client->Mithril: Initialise module Mithril->Server: Data query Server->Mithril: Data result Mithril-->Client: Render page with UI events and data

Client page lifecycle

The lifecycle of the page in SPA mode runs mostly on the client side - only data is fetched from the server.

Participant Client Participant Mithril Participant Server Note over Client: At /users we click\n "edit user" link Client->Mithril: GET /user/123 Note over Mithril: Runs user.edit MVC component\non the client only Mithril-Server: Data query Server->Mithril: Data result Mithril-->Client: Render the user edit page
